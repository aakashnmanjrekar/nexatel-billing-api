NexaTel Mock Billing API

A simulated telecom billing REST API built for the Sprinklr Invoice Explanation Bot. Designed to be the external endpoint Sprinklr Webhooks call instead of a real billing system.

📡 Endpoints

GET /health Service health check – no auth required 
{
  "status": "ok",

  "version": "1.0.0",
  "timestamp": "2026-05-18T10:30:00.000Z"
}
GET /billing/v1/lookup?mobile=:mobile Resolve mobile number → Account ID Bot Auth Step

Called after OTP verification to resolve the customer's mobile number to their Account ID, which is stored in CF_Account_ID.

ParamInDescription
mobile*queryCustomer mobile number (e.g. 0612345678)
GET /billing/v1/lookup?mobile=0612345678
X-API-Key: nexatel-test-key

// Response
{
  "account_id": "NXT-001",
  "mobile": "0612345678"
}
GET /billing/v1/invoices/:account_id Fetch current month's invoice WH_Fetch_Current_Invoice

Returns the current month invoice when no month/year params are passed.

ParamInDescription
account_id*pathCustomer account ID (e.g. NXT-001)
GET /billing/v1/invoices/NXT-001
X-API-Key: nexatel-test-key

// Response – maps to CF_Current_Month_Invoice
{
  "account_id": "NXT-001",
  "billing_period": "2026-05",
  "total_amount": 85.40,
  "currency": "EUR",
  "due_date": "2026-06-15",
  "status": "Due",
  "line_items": [
    { "category": "Base Plan",           "amount": 29.99 },
    { "category": "International Calls", "amount": 32.50 },
    { "category": "Data",                "amount": 15.00 },
    { "category": "Taxes & Fees",       "amount": 7.91  }
  ]
}
GET /billing/v1/invoices/:account_id?month=&year= Fetch invoice for specific month WH_Fetch_Previous_Invoice

Used by the comparative intent to fetch the previous month. Pass month=04&year=2026 or shorthand month=2026-04.

ParamInDescription
account_id*pathCustomer account ID
month*queryMonth number (04) or YYYY-MM (2026-04)
yearoptqueryYear (2026) — required if month is not YYYY-MM
GET /billing/v1/invoices/NXT-001?month=04&year=2026
X-API-Key: nexatel-test-key

// Response maps to CF_Previous_Month_Invoice
{
  "billing_period": "2026-04",
  "total_amount": 47.90,
  ...
}
GET /billing/v1/invoices/:account_id?from=&to= Fetch invoices for a date range WH_Fetch_Invoice_Range

Used by Invoice_History intent. Returns all invoices between from and to periods (inclusive).

ParamInDescription
from*queryStart period in YYYY-MM format (e.g. 2026-01)
to*queryEnd period in YYYY-MM format (e.g. 2026-03)
GET /billing/v1/invoices/NXT-001?from=2026-01&to=2026-03
X-API-Key: nexatel-test-key

// Response
{
  "account_id": "NXT-001",
  "from": "2026-01",
  "to": "2026-03",
  "invoice_count": 3,
  "invoices": [...]
}
GET /billing/v1/invoices/:account_id/lineitems Detailed line item breakdown with charge highlighting WH_Fetch_LineItem_Detail

Used by Invoice_LineItem intent. Pass charge_type to flag the queried line item in the response.

ParamInDescription
charge_typeoptqueryLine item to highlight (e.g. International Calls, Roaming)
monthoptqueryMonth — defaults to current
yearoptqueryYear — defaults to current
GET /billing/v1/invoices/NXT-001/lineitems?charge_type=International%20Calls
X-API-Key: nexatel-test-key

// Response includes highlighted flag on matched item
{
  "queried_charge_type": "International Calls",
  "line_items": [
    { "category": "Base Plan",           "amount": 29.99, "highlighted": false },
    { "category": "International Calls", "amount": 32.50, "highlighted": true  },
    ...
  ]
}

⚙️ Sprinklr Webhook Configuration

Configure these in Sprinklr → Settings → Webhook Connectors. Replace YOUR-DOMAIN with your deployed URL.

WH_Fetch_Current_Invoice
GET https://YOUR-DOMAIN/billing/v1/invoices/{{CF_Account_ID}}
Trigger: All intents except Invoice_History
WH_Fetch_Previous_Invoice
GET https://YOUR-DOMAIN/billing/v1/invoices/{{CF_Account_ID}}?month={{prev_month}}&year={{prev_year}}
Trigger: Invoice_Comparative intent only
WH_Fetch_Invoice_Range
GET https://YOUR-DOMAIN/billing/v1/invoices/{{CF_Account_ID}}?from={{start_period}}&to={{end_period}}
Trigger: Invoice_History intent
WH_Fetch_LineItem_Detail
GET https://YOUR-DOMAIN/billing/v1/invoices/{{CF_Account_ID}}/lineitems?charge_type={{entity_charge_type}}
Trigger: Invoice_LineItem intent
WH_Customer_Lookup (Auth)
GET https://YOUR-DOMAIN/billing/v1/lookup?mobile={{verified_mobile}}
Trigger: OTP verification success → maps account_id → CF_Account_ID
